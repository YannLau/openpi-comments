  为什么 delta 动作看起来也是"连贯"的？

  这是你真正困惑的地方。Delta 的公式是：

  delta_actions[t] = target_absolute[t] - state   (对 mask=True 的维度)

  注意 state 是同一个当前时刻的状态，不是每个时间步各有不同的 state。所以：

  delta_actions[0] = target_absolute[0] - state
  delta_actions[1] = target_absolute[1] - state
  delta_actions[2] = target_absolute[2] - state
  ...

  因为 state 对所有时间步是同一个常数，所以 delta
  序列的变化趋势和绝对值序列完全一样——只是整体平移了一个 state 的偏移：

  delta[t] - delta[t+1] = (target[t] - state) - (target[t+1] - state) =
  target[t] - target[t+1]

  delta 的"连贯性"和绝对值的"连贯性"完全等价。你不能通过"看起来连不连贯"来区分，
  而应该看数值量级——True 的值都在 ±0.05 以内，False 有 -1.58 的大值。

  actions[..., :dims] -= np.expand_dims(np.where(mask, state[..., :dims], 0),

  delta[t] = a[t] - state      ← 所有时间步 shared state
  a[t] = delta[t] + state

● 好问题。state 指的就是数据样本中当前时刻的机器人关节角度。

  打开 LeRobot 数据集来看

  当你从 LeRobot 数据集取一个样本时，它返回的数据结构大概是这样：

  {
      "observation.state": [0.044, -0.0, 0.001, -1.571, 0.236, 0.045, ...],  # 
  当前关节位置
      "action":  [                                        # 未来 50 步的目标位置
          [0.023, 0.237, -0.008, -1.584, 0.236, 0.031, ...],   # action[0] = 
  下一帧目标
          [0.020, 0.238, -0.007, -1.583, 0.236, 0.027, ...],   # action[1]
          ...
      ]
  }

  state 就是 observation.state——在记录这个样本时，机器人关节当前的实际角度。

  换算验证

  关节 0: state = a[0] - d[0] = 0.02306 - (-0.02104) = 0.04410

  这个 0.04410 弧度 ≈ 2.53°，就是：
  - 数据被记录时，机械臂关节 0 的当前实际角度
  - 它是一个角度值（弧度），来自机器人的编码器读数

  核心理解

  ┌───────────┬────────────────────────┬─────────────────────────────────┐
  │    量     │          含义          │         值域（关节 0）          │
  ├───────────┼────────────────────────┼─────────────────────────────────┤
  │ state     │ 当前时刻的关节位置     │ 0.044 rad                       │
  ├───────────┼────────────────────────┼─────────────────────────────────┤
  │ action[t] │ 第 t 帧的目标绝对位置  │ 0.023 → 0.020 → 0.017... rad    │
  ├───────────┼────────────────────────┼─────────────────────────────────┤
  │ delta[t]  │ 从当前位置到目标的偏移 │ -0.021 → -0.024 → -0.027... rad │
  └───────────┴────────────────────────┴─────────────────────────────────┘

  关系很简单：delta[t] = action[t] - state（对 mask=True 的维度）

  所以：
  - 如果你用 True，模型学的是**"我从当前位置移多少"**
  - 如果你用 False，模型学的是**"目标位置在哪里"**

  两者训练的效果理论等价——因为推理时 AbsoluteActions 又加回来 state。选择 delta
  通常是为了让模型学一个更稳定的表示（偏移量比绝对值更容易泛化）。
